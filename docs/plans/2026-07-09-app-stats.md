# App Stats Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Track per-app "open" and "view" events as daily rollups, and surface totals/trends to owning developers (MyApps) and admins (AdminView).

**Architecture:** A new `app_stats_daily` table (one row per app per day, `opens`/`views` counters) is upserted by a public, rate-limited `POST /api/apps/{slug}/track` endpoint. Reads are gated by a new `ownerOrAdminAuth` wrapper (`GET /api/apps/{slug}/stats`) or folded into the existing admin apps listing (`total_opens`/`total_views` columns). Frontend fires beacons from the existing open/copy-link/detail-load code paths, and renders numbers plus a hand-rolled SVG sparkline — no new chart dependency.

**Tech Stack:** Go backend with PostgreSQL migrations, OpenAPI YAML generation, Vue 3 frontend, Vitest.

**Files:**
- Create: `backend/migrations/016_app_stats.sql`

- Modify: `backend/handlers.go` (add `statsLimiter *rateLimiter` to `server` struct)
- Modify: `backend/main.go` (wire `POST /api/apps/{slug}/track`, init `statsLimiter`)
- Test: `backend/stats_test.go`

**Steps:**
1. Add the migration: `app_stats_daily(app_id UUID REFERENCES apps(id) ON DELETE CASCADE, day DATE, opens INT NOT NULL DEFAULT 0, views INT NOT NULL DEFAULT 0, PRIMARY KEY (app_id, day))`.
2. Add failing tests in `stats_test.go`: `trackEvent` upserts a new day-row, increments an existing one, rejects an unknown `event` value, and no-ops (200/204, no panic) for an unknown slug.
3. Add failing test for the rate limit: reuse the existing `rateLimiter` (`backend/reviews.go`) — 20 requests/minute per `ip+slug` key; 21st call within the window is rejected.
4. Implement `trackEvent(ctx, pool, appID, event string) error` doing the upsert (`INSERT ... ON CONFLICT (app_id, day) DO UPDATE SET opens = app_stats_daily.opens + 1` or `views = app_stats_daily.views + 1`).
5. Implement `(s *server) trackApp(w, r)`: look up the app id by slug (reuse `s.fetchApp` or a lighter id-only lookup), read `{"event": "open"|"view"}` from the body, check `s.statsLimiter.allow(r.RemoteAddr + ":" + slug)`, call `trackEvent`, always respond `204` (even on not-found/bad-event, per design — a lost stat ping is not user-facing breakage) except for the rate-limit case which returns `429`.
6. Register `mux.HandleFunc("POST /api/apps/{slug}/track", s.trackApp)` in `main.go`, and add `statsLimiter: newRateLimiter(20, time.Minute)` to the `server{}` construction next to `reviewLimiter`.
7. Run `go test ./backend/... -run Stats`.
8. Commit.

### Task 2: Stats Read Endpoint (Owner Or Admin)

**Files:**
- Modify: `backend/stats.go`
- Modify: `backend/adminauth.go` (add `ownerOrAdminAuth` helper)
- Modify: `backend/main.go` (wire `GET /api/apps/{slug}/stats`)
- Test: `backend/stats_test.go`

**Steps:**
1. Add failing tests: `GET /api/apps/{slug}/stats` returns `401` with no cookie/token, `200` for the app's owner wallet, `200` for an admin wallet, `200` for a valid admin bearer token, `403` for a wallet that owns a *different* app.
2. Implement `(s *server) ownerOrAdminAuth(slug string, next func(w http.ResponseWriter, r *http.Request)) http.HandlerFunc` in `adminauth.go`: check the wallet cookie first — if `verifyWalletCookie` succeeds and (`isAdminWallet(s.adminWallets, address)` OR `s.isOwner(ctx, slug, address)`) is true, call `next`; else fall back to the existing admin bearer-token comparison from `adminAuthMiddleware`; else `401`.

   Note: `slug` comes from `r.PathValue("slug")`, so this wrapper reads the path value itself rather than taking it as a pre-bound argument — call it as `s.ownerOrAdminAuth(s.appStats)` where `appStats` re-reads `r.PathValue("slug")` internally.
3. Implement `(s *server) appStats(w, r)`: read `slug := r.PathValue("slug")`, look up the app id, query `app_stats_daily` for the last 30 days ordered by `day`, sum for totals, and write:
   ```json
   { "totals": { "opens": 812, "views": 2044 },
     "daily": [{ "date": "2026-06-10", "opens": 12, "views": 30 }] }
   ```
   Missing days are omitted (not zero-filled) — the frontend sparkline fills gaps.
4. Register `mux.HandleFunc("GET /api/apps/{slug}/stats", s.ownerOrAdminAuth(s.appStats))` in `main.go`.
5. Run `go test ./backend/... -run Stats`.
6. Commit.
